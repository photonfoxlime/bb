# Code Quality Backlog -- 2026-03-07

## High Priority -- Bugs / Correctness

### 1. ~~Write errors mapped to `MountError::Read`~~ FIXED
- **Location:** `src/store/mount.rs:535,542,547`
- **Issue:** In `save_subtree_to_file`, `fs::create_dir_all` and `fs::write` failures are mapped to `MountError::Read`, producing misleading messages like "failed to read mount file..." for write failures.
- **Fix:** Use `MountError::Write`.

### 2. ~~Collapse/Expand share the same i18n key~~ FIXED
- **Location:** `src/app/action_bar.rs:132-133`
- **Issue:** Both `ActionId::CollapseBranch` and `ActionId::ExpandBranch` map to `"action_expand_branch"`. Collapse tooltip displays the expand label.
- **Fix:** Create a separate `"action_collapse_branch"` key in all three locale files.

- **Location:** `locales/ja.yml`, `locales/zh-CN.yml`

- **Fix:** Add translated entries.

### 4. ~~`theme::LIGHT.accent_muted` used unconditionally in settings~~ FIXED
- **Location:** `src/app/settings.rs` (11 occurrences)
- **Issue:** Hardcodes the light palette for label colors. In dark mode, labels render with incorrect light-mode colors.
- **Fix:** Use theme-aware resolution by threading `palette` through helpers.

### 5. ~~`save()` silently succeeds when path is unavailable~~ FIXED
- **Location:** `src/store/persist.rs:47-49`
- **Issue:** When `AppPaths::data_file()` returns `None`, `save()` returns `Ok(())` without writing or logging.
- **Fix:** Added `tracing::warn!`.
